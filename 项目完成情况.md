# Desktop Agent 项目完成情况

## 已完成模块概览

```
src/
├── schemas/          ★ 核心数据类型
├── utils/            ★ 配置/密钥/日志/LLM工厂/用户设置
├── tools/            ★ 18 个工具（桌面/浏览器/AI） + 注册中心
├── agent/            ★ Agent 核心引擎（规划/执行/验证/记忆/主循环）
├── perception/       ★ 感知层（截图/UIA解析）
├── platforms/        ★ 平台编排（WPS/知乎/微信）
├── ui/               ★ Gradio Web UI（任务页 + 设置页 + 事件系统）
└── main.py           ★ CLI 入口（6 种运行模式）
```

## 一、核心引擎（src/agent/）

实现了完整的 **Plan → Execute → Observe → Verify** Agent 闭环。

| 模块 | 状态 | 说明 |
|------|------|------|
| `planner.py` | 完成 | LLM 将自然语言目标分解为有序工具调用序列，失败时支持 re-plan |
| `executor.py` | 完成 | 从 ToolRegistry 查找工具、执行、返回结构化 ActionResult |
| `verifier.py` | 完成 | 规则检查 + LLM 视觉/文本双重验证 |
| `observer.py` | 完成 | 聚合 DOM/UIA/截图等感知源，生成统一 Context |
| `memory.py` | 完成 | 三层记忆：短期（滑动窗口）、工作（会话状态）、长期（ChromaDB）|
| `state.py` | 完成 | 会话级可变状态（浏览器页面、窗口句柄、登录态等）|
| `loop.py` | 完成 | 主循环：规划→执行→验证→失败重试→重新规划，集成 EventBus 实时推送 |

## 二、工具层（src/tools/）— 18 个已注册工具

### 桌面工具（9 个）
基于 **COM 接口**（pywin32）操作 WPS/Word：

| 工具 | 功能 |
|------|------|
| `launch_app` | 启动 WPS/Word/微信/记事本，含注册表路径查找 |
| `close_app` | 关闭应用窗口 |
| `create_document` | 新建空白文档 |
| `write_document_text` | 写入文字（支持标题/正文样式）|
| `set_font` | 设置字体、字号、加粗 |
| `set_alignment` | 设置段落对齐（左/中/右/两端）|
| `save_document` | 保存为 .docx 文件 |
| `export_pdf` | 导出为 PDF |
| `insert_image` | 在光标处插入图片 |

### 浏览器工具（6 个）
基于 **Playwright**：

| 工具 | 功能 |
|------|------|
| `navigate` | 导航到 URL |
| `click` | 点击元素（支持 CSS/文本/role 三种定位）|
| `type_text` | 输入文本 |
| `screenshot` | 页面/元素截图，返回 base64 |
| `get_dom` | 获取精简 DOM 摘要（仅可交互元素）|
| `extract_text` | 提取页面可见文本 |

### AI 工具（3 个）
| 工具 | 功能 |
|------|------|
| `generate_article` | LLM 生成结构化文章（支持主题/风格/篇幅/大纲）|
| `summarize` | 文本摘要 |
| `analyze_screen` | 截图内容分析 |

### 工具基础设施
- `BaseTool` — 抽象基类（execute/validate/safe_execute/to_openai_tool）
- `ToolRegistry` — 按名注册、查找、导出 OpenAI function-calling 格式

| `zhihu.py` | 知乎文章发布任务构建器 |

## 五、Web UI（src/ui/）

### 事件系统（`events.py`）
- `AgentEvent` — 10 种事件类型（plan_start/done、step_start/done/retry、screenshot、error、task_done）
- `EventBus` — 异步发布/订阅，支持历史回放

### Gradio 界面（`gradio_app.py`）
- **任务页**：自然语言输入 → 实时展示步骤卡片（待执行/执行中/成功/失败/重试）+ 截图 + 控制台
- **设置页**：Provider 选择（DeepSeek/OpenAI/Anthropic）、密钥输入（密码框+显示切换）、模型选择
- **UIBridge**：后台线程运行 AgentLoop，通过队列线程安全推送事件到 UI

## 七、数据类型（src/schemas/）

| 类型 | 说明 |
|------|------|
| `Task` / `Step` / `ActionResult` | 任务与步骤生命周期 |
| `TaskStatus` / `RetryPolicy` | 状态与重试枚举 |
| `UIElement` / `Selector` / `PerceptionResult` | 跨平台 UI 元素与感知结果 |

## 八、入口（src/main.py）— 6 种运行模式

```bash
python src/main.py "任务描述"          # CLI 单次任务
python src/main.py -i                  # REPL 交互模式
python src/main.py --ui                # Gradio Web UI
python src/main.py --wps "主题"        # 快捷：WPS 文档创作
python src/main.py --zhihu "主题"      # 快捷：知乎发文
python src/main.py --wechat "账号名"   # 快捷：微信关注
```
